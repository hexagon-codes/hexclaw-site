Privacy Policy

HexClaw is local-first. When you use a configured cloud model, necessary content is sent to that third-party provider.

Data Storage

Conversations, settings, and local indexes are stored on your device by default. hexagon-codes does not provide an official cloud account service that syncs or hosts this data. When you actively use a third-party provider, input needed for that request is sent to the endpoint you configured.

API Key Security

The desktop app keeps API keys in dedicated local storage, separate from ordinary configuration. Browser development mode keeps them only in the current page's memory, so they must be entered again after a refresh. Use device sign-in protection and disk encryption, and do not share the device account.

Network Communication

HexClaw communicates only with network services you configure or enable. For the content a third-party model may receive, knowledge-base text chunks, query text, and your controls, see the Third-Party AI Services & Data Notice.

Fully Offline Mode

When you use only local models such as Ollama and disable other network integrations, HexClaw can run completely offline and those model requests stay on your device.

Data Collection and Tracking

We do not collect user data, track user behavior, or serve ads. HexClaw is open-source software; you may audit the code at any time to verify these commitments.

Open Source and Transparency

HexClaw is released under the Apache-2.0 license. The full source code is available on GitHub for review. We welcome community participation in code audits and contributions.
