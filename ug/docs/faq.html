ھېشى AI ھۆججەتلەر

كۆپ كۆرۈلىدىغان سوئاللار

جەملىمەھېشى AI ئۈستەلئۈستى ئۇچى在ئورنىتىشقوزغىتىش، مودېلتەڭشەش، سانلىق-مەلۇماتشەخسىيەت، ئىش ئېقىمىۋە MCP ئۇلىنىشتىكىكۆپ كۆرۈلىدىغان سوئاللاربىلەنخاتالىق تەكشۈرۈش ئۇسۇلى.

ئورنىتىشبىلەنقوزغىتىش

Q: ئەپقوزغىتىش后白屏/无法加载?

Go Sidecar موتوربولىدۇ能未نورمالقوزغىتىش. تەكشۈرۈش步骤:

  1. كۆرۈش خاتىرە بەت是否有错误ئۇچۇر
  2. جەزملەش端口 16060 未被占用
  3. 尝试قايتا قوزغىتىشئەپ، Sidecar بولىدۇ自动在قوزغىتىش时拉起 (最多قاتارلىق待 30 秒)

Q: macOS ئەسكەرتىش "无法ئېچىش، 因ئۈچۈن无法验证ئىجادكار"?

右键چېكىشئەپ → "ئېچىش"، 在弹出的سۆھبەت框中再次چېكىش "ئېچىش". ياكى者ئىجرا قىلىش: 

xattr -cr /Applications/HexClaw.app

Q: 如何开机自قوزغىتىش?

كىرىش تەڭشەك → 通用 → 开机自قوزغىتىش، 开启即بولىدۇ. macOS ئىشلىتىش Launchئاجانت 实现.

مودېلبىلەنسۆھبەت

Q: API 请求قايتىش 401 / 403 错误?

API Key 无效ياكى过期. كىرىش تەڭشەك → مودېل مۇلازىمەت تەمىنلىگۈچى، تەكشۈرۈش Key 是否正确، جەزملەش账户余额充足.

Q: 如何ئىشلىتىشيەرلىك مودېل (不联网) ?

ئورنىتىش Ollama 后، 在ھېشى AI 中قوشۇش Ollama مۇلازىمەت تەمىنلىگۈچى:

  • Base URL: http://localhost:11434/v1
  • API Key: 留空
  • مودېل: 手动قوشۇش已چۈشۈرۈش的مودېل名 (如 llama3.1، qwen2.5)

Q: ئېقىپ چىقىرىش卡住不动了?

بولىدۇ能是网络超时. 按 Esc 停止生成، 重新发送ئۇچۇر即بولىدۇ.

سانلىق-مەلۇماتبىلەنشەخسىيەت

Q: 我的سانلىق-مەلۇمات存在哪里?

بارلىقسانلىق-مەلۇماتساقلانغان本地:

  • كۆرۈشۈشۋەئۇچۇر: SQLite سانلىق-مەلۇمات库 (hexclaw.db)
  • تەڭشەك: Tauri Store (本地 JSON ھۆججەت)
  • API Key: 操作سىستېمابىخەتەرلىك密钥库
  • سانلىق-مەلۇماتمۇندەرىجە: بولىدۇ在 تەڭشەك → 通用 → سانلىق-مەلۇماتمۇندەرىجە كۆرۈش

Q: سانلىق-مەلۇماتبولىدۇ发送到第三方مۇلازىمېتىر吗?

不بولىدۇ. ھېشى AI 仅بىلەن你تەڭشەش的 AI API 端点通信. 如果ئىشلىتىش Ollama يەرلىك مودېل، بارلىقسانلىق-مەلۇمات完全离线.

ئىش ئېقىمىبىلەن ئاجانت

Q: ئىش ئېقىمىئىجرا قىلىش失败了怎么办?

كۆرۈش失败节点的错误ئۇچۇر. 常见原因: ئاجانت 未تەڭشەشمودېل، MCP قورالئۇلىنىش断开، Token 超限.

Q: 多 ئاجانت بولىدۇ议最多قوللاش几个 ئاجانت?

没有硬性限制، 但تەكلىپ 2~5 个 ئاجانت 参بىلەن، 过多بولىدۇ导致سۆھبەت变长، Token 消耗增大.

MCP بىلەنقورال

Q: MCP مۇلازىمېتىرئۇلىنىش失败?

  1. جەزملەشمۇلازىمېتىر进程已قوزغىتىش
  2. تەكشۈرۈشكېلىشىم类型是否匹配 (stdio / sse / streamable_http)
  3. كۆرۈش خاتىرە بەت获取详细错误ئۇچۇر

深度研究

深度研究ھالەت一直停在"ئىزدەش"阶段怎么办?

جەزملەش ئاجانت بولىدۇ访问网络ئىزدەشقورال (Skill 中的 web_search ئېھتىياج已启用) . 如果كەينى ئۇچ没有تەڭشەشئىزدەش Skill، 研究ھالەتبولىدۇ退化ئۈچۈن普通 ئاجانت 回复.

研究报告质量不高?

尝试:

  • ئىشلىتىش更强的مودېل (如 Claude Sonnet 4 ياكى GPT-4o)
  • 提供更明确的研究主题ۋە范围
  • 在输入中指定输出格式要求 (如"生成带引用的 Markdown 报告")

IM ئېقىم

تەڭشەش了飞书/钉钉/企微، 但收不到ئۇچۇر?

  1. جەزملەشتەڭشەشساقلاش后已قايتا قوزغىتىش Engine
  2. تەكشۈرۈش App Secret / Token 是否正确 (دىققەت区分سىناش环境ۋە正式环境的凭证)
  3. جەزملەش机器人已قوشۇش到目标群组ياكى已启用ئۇچۇر接收ھوقۇق
  4. كۆرۈشخاتىرە页是否有ئۇلىنىش错误ئۇچۇر

ماھارەت بازىرى

ClawHub ماھارەتئورنىتىش后不生效?

当前 ClawHub 处于 Mock ھالەت (本地ئالدىن كۆرۈش) ، ئورنىتىش操作不بولىدۇ真正چۈشۈرۈشماھارەتھۆججەت. 正式版上线后将直接对接 ClawHub جامائەت API. 本地ئورنىتىش的 .md Skill ھۆججەتبولىدۇنورمالئىشلىتىش.

性能بىلەنئەلالاشتۇرۇش

Q: ئەپ占用内存较高?

ھېشى AI ئىشلىتىش WebView 渲染كۆرۈنمە يۈز. نورمال内存占用约 200~400MB. 如果超过 1GB، 尝试:

  • 关闭不必要的كۆرۈشۈشبەلگە
  • 清理خاتىرە缓存 (خاتىرەبەت → 清除)
  • قايتا قوزغىتىشئەپ

Q: 如何كۆرۈشموتورئىجرا قىلىش状态?

仪表盘بەت显示 Sidecar موتور健康状态، 包括ئۇلىنىش状态، API 端口قاتارلىقئۇچۇر. موتور每 5 秒自动检测一次.

بىلەن其他مەھسۇلات的区别

Q: HexClaw ۋە ChatGPT Desktop / Claude Desktop 有什么不同?

ChatGPT Desktop ۋە Claude Desktop 是特定مۇلازىمەت تەمىنلىگۈچى的官方خېرىدار، 只能ئۇلىنىش自家مودېل. HexClaw 是مودېل无关的 —— 你بولىدۇ同时ئۇلىنىش OpenAI، Claude، DeepSeek، Gemini، Ollama قاتارلىقبىرنەچچەمۇلازىمەت تەمىنلىگۈچى، 并在同一个كۆرۈنمە يۈز中ئالماشتۇرۇش. 此外، HexClaw 内置了 ئاجانت موتور، MCP قورالكېلىشىم، ئىش ئېقىمى تاختىسىۋە RAG بىلىم ئامبىرى، بولسا完整的 AI ئاجانت ئىجرا قىلىش平台، 而不仅仅是سۆھبەتخېرىدار.

Q: HexClaw ۋە Cursor 有什么不同?

Cursor بولسا AI 代码تەھرىرلەش器، 专注于编程ئەھۋال. HexClaw 定位是通用 AI ئاجانت ئۈستەلئۈستى خېرىدار، قاپلاشسۆھبەت، ئاجانت تەرتىپلەش، ئىش ئېقىمى自动化، بىلىم ئامبىرىقاتارلىق多种ئەھۋال، 不限于代码生成.

سانلىق-مەلۇماتباشقۇرۇش

Q: 如何زاپاس/迁移سانلىق-مەلۇمات?

HexClaw 的بارلىقسانلىق-مەلۇماتساقلانغان本地سانلىق-مەلۇماتمۇندەرىجە中 (بولىدۇ在 تەڭشەك → 通用 → سانلىق-مەلۇماتمۇندەرىجە كۆرۈش) . زاپاس时只需كۆچۈرۈش该مۇندەرىجە. 迁移到新电脑时، 将سانلىق-مەلۇماتمۇندەرىجەكۆچۈرۈش到对应位置即بولىدۇ.

Q: بىلىم ئامبىرىقوللاش多大的ھۆججەتلەر?

单个ھۆججەتلەرتەكلىپ不超过 50MB. بىلىم ئامبىرى总容量取决于本地磁盘空间. ھۆججەتلەر上传后بولىدۇ自动分块 (chunking) ۋە向量嵌入، 检索效率不受ھۆججەتلەر总量影响.
