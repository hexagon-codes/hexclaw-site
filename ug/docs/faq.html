كۆپ كۆرۈلىدىغان سوئاللار

ئورنىتىش، ئۇلىنىش، يەرلىك مودېل، سانلىق-مەلۇمات ۋە ئىشلىتىشتىكى كۆپ كۆرۈلىدىغان مەسىلىلەر.

ئورنىتىش

ئەپ ئېچىلماي قالسا قانداق قىلىمەن؟

macOS دا Gatekeeper ئاگاھلاندۇرۇشى كۆرۈلسە، سىستېما تەڭشەكلىرىدىكى بىخەتەرلىك بۆلىكىدىن ئېچىشقا رۇخسەت قىلىڭ. Windows دا قاچىلاش بولىقىنى قايتا چۈشۈرۈپ، ۋىرۇسخور توسۇۋاتامدۇ تەكشۈرۈڭ.

Homebrew دىن كېيىن سۆھبەت ئىشلىمىسە؟

ئەپنى قايتا قوزغىتىڭ، خاتىرە بېتىدە Engine نىڭ قوزغالغانلىقىنى تەكشۈرۈڭ، ئاندىن مودېل تەمىنلىگۈچىگە ئۇلىنىش سىنىقى قىلىڭ.

مودېل ۋە API

API Key قەيەردە ساقلىنىدۇ؟

API Key يەرلىك مەشغۇلات سىستېمىسىنىڭ بىخەتەر ئاچقۇچ ساقلاش مۇلازىمىتىدە ساقلىنىدۇ.

Ollama نى قانداق ئىشلىتىمەن؟

Ollama نى ئورنىتىپ مودېل چۈشۈرۈڭ، ھېشى AI دا Ollama تەمىنلىگۈچىنى قوشۇڭ، base URL نى http://127.0.0.1:11434 قىلىپ ساقلاڭ، API Key نى بوش قالدۇرۇڭ.

سانلىق-مەلۇمات

ئۇچۇرلىرىم قەيەردە؟

كۆرۈشۈش، ئۇچۇر، تەڭشەك ۋە بىلىم ئامبىرى سانلىق-مەلۇماتلىرى يەرلىك ئۈسكۈنىدە ساقلىنىدۇ.

سانلىق-مەلۇمات ئۈچىنچى تەرەپكە ئەۋەتىلەمدۇ؟

ياق. پەقەت سىز سەپلىگەن AI API endpoint قا تەلەپ ئەۋەتىلىدۇ. Ollama يەرلىك مودېل ئىشلىتىلسە، مەزمۇن تورغا چىقمايدۇ.

ئىجرا قىلىش ۋە قوراللار

MCP ئۇلانمىسا؟

  • MCP مۇلازىمېتىر جەريانى قوزغالغانلىقىنى تەكشۈرۈڭ.
  • stdio، sse ياكى streamable_http كېلىشىم تۈرى توغرىلىقىنى جەزملەشتۈرۈڭ.
  • خاتىرە بېتىدىن تولۇق خاتالىق ئۇچۇرىنى كۆرۈڭ.

ئىش ئېقىمى مەغلۇپ بولسا؟

مەغلۇپ بولغان تۈگۈننىڭ خاتالىق ئۇچۇرىنى كۆرۈڭ. كۆپ ئۇچرايدىغان سەۋەبلەر: Agent قا مودېل بەلگىلەنمىگەن، قورال ھوقۇقى يېتەرلىك ئەمەس، MCP ئۈزۈلگەن ياكى Token چېكىدىن ئېشىپ كەتكەن.

ئىقتىدار

ئەستە تۇتۇش كۆپ ئىشلىتىلسە ئەپ ئاستىلامدۇ؟

ئادەتتە يوق. ئەمما ناھايىتى چوڭ كۆرۈشۈش ياكى ھۆججەتلەر بىلەن ئىشلىگەندە، كونا كۆرۈشۈشلەرنى ئارخىپلاش ۋە بىلىم ئامبىرىنى تەرتىپلەش تەۋسىيە قىلىنىدۇ.
