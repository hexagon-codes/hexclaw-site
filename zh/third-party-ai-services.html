第三方 AI 服务与数据说明

河蟹 AI 本地优先。只有当你选择云端模型或使用会调用云端的“自动”方案时，相关请求才会发送给你配置的第三方 Provider。

先说结论：云端模型由你配置的第三方 Provider 提供。河蟹 AI 不代售模型服务，也不会替你创建 Provider 账户；Provider 的费用、可用性和数据规则由其条款决定。你随时可以改用本地模型或纯文本检索。

1. 这份说明适用于什么

本页说明河蟹 AI 桌面端将请求交给第三方 AI Provider 时的产品行为，包括聊天、语义索引和语义检索。它是对《隐私政策》与《使用条款》的补充。

河蟹 AI 是本地优先的开源客户端。仅仅分发客户端，并不意味着 hexagon-codes 自动成为你所选 Provider 的运营方或代理商；同时，任何主体的法律角色与责任仍应依据实际部署方式和所提供的功能、对处理目的和方式的控制等事实判断，不能只用“工具”或“壳”来概括。

2. 什么时候会连接第三方 Provider

你的选择会发生什么
本地模型模型推理在你配置的本地运行时完成，该次模型请求不会发送给第三方云端 Provider。
云端模型在你触发对应操作后，客户端将必要内容和凭据发送到你配置的 Provider Endpoint。
自动客户端按当时可用性与策略选择模型；如果实际选择云端模型，会按云端数据路径处理，并在界面显示实际使用的模型。
纯文本检索只使用本地已提取文本与全文索引，不为该次检索调用 Embedding Provider。

“已配置”或“可选”不等于正在传输。通常只有在你提交请求、开始索引、重新索引或执行语义查询时才会发起对应网络请求。网络延迟、Provider 排队、限流和配额都会影响云端速度。

3. 知识库语义索引会发送什么

在当前知识库 Embedding 链路中，PDF 等文档先在本地提取与分块。使用云端 Embedding 时，发送的是规范化文本分块；执行语义检索时，发送的是查询文本。Embedding 请求不包含原始 PDF 文件本体、原始文件名、本地路径或本地全文索引数据库。

文本分块本身仍可能包含个人信息、商业秘密或其他敏感内容。请在选择云端模型前判断内容是否适合交给该 Provider。聊天、图片理解或你主动添加附件等其他功能，可能按功能需要发送提示词、上下文、图片或附件；它们不受上一段“仅文本分块”的范围描述所覆盖。

4. Provider 会收到哪些信息

  • 完成所选功能所必需的输入，例如提示词、上下文、规范化文本分块、查询文本或你主动提供的附件；
  • 模型名、请求参数以及服务运行所需的技术信息；
  • 你配置的 API Key 或其他认证信息，直接用于访问对应 Endpoint。

Provider 可能依据自身规则处理请求日志、账户信息、设备或网络信息。其计费、服务可用性、数据留存、是否用于模型训练、处理地区、安全措施与删除机制，均以该 Provider 向你提供的最新条款、隐私政策和控制台设置为准。

5. 你拥有的选择与控制

  • 在语义索引设置中选择本地、云端或自动模型，并查看模型的“本地 / 云端”标识；
  • 处理不宜出端的资料时使用本地模型，或先使用纯文本检索；
  • 在设置中移除或替换 Provider Endpoint 与 API Key；
  • 删除本地知识库、索引或会话数据；如需删除 Provider 留存的数据，应通过该 Provider 的控制台或权利请求渠道办理；
  • 在发送前对文档进行脱敏、最小化，并避免上传与目的无关的内容。

6. 未成年人和儿童个人信息

在中国境内，未满 14 周岁未成年人的个人信息属于敏感个人信息。依法处理此类信息，应取得父母或其他监护人同意，并制定专门的个人信息处理规则；还应遵循目的明确、最小必要和安全保障等要求。

如果你是父母、监护人、教师或学校管理者，请由成年人配置模型与 Provider，并在发送前移除与任务无关的姓名、联系方式、健康情况、身份信息、面部图像和学习档案。不要让未成年人将 API Key 或敏感资料直接提交给不受信任的 Endpoint。

本页用于说明产品数据流，不能替代依法需要履行的告知、同意、单独同意、监护人同意、安全评估或专门规则等义务。实际部署者或运营者应根据其处理目的、对象、场景和适用地区独立评估并采取措施。

7. 跨境传输与处理地区

如果你配置的 Provider Endpoint 或其基础设施位于境外，请求可能构成或涉及个人信息跨境传输。请在启用前核对 Provider 的处理地区、分包商、数据驻留选项与跨境机制，并根据适用法律判断是否需要告知、单独同意、标准合同、认证或安全评估。河蟹 AI 不会替你选择 Provider 的处理地区。

8. 安全、异常与事件处理

  • API Key 仅应保存在受你控制的设备中，避免共享、截图或提交到代码仓库；
  • 使用 HTTPS Endpoint，并定期轮换、限制权限与检查 Provider 账单；
  • 发现异常调用时，立即在 Provider 控制台撤销 Key，并检查本地日志和账户活动；
  • 若问题涉及 Provider 的服务或数据事件，请同时联系该 Provider；若涉及河蟹 AI 客户端，请联系 support@hexclaw.net。

9. 服务边界与法定权利

第三方模型的输出可能不准确、有偏差或不适合特定用途。你应核验重要结论，不应将未经复核的输出直接用于医疗、法律、金融、安全或其他高风险决策。

开源软件许可证中的“按原样”条款，以及 Provider 自身条款中的责任安排，均应在适用法律允许的范围内理解。任何依法不能排除或限制的责任，以及消费者、个人信息主体和未成年人的法定权利，不因本页说明而被排除或限制。
